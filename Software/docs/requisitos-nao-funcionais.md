# Elicitação de Requisitos Não Funcionais

1. A transferência das instruções não pode ultrapassar 10 segundos.  
2. A consolidação do trajeto realizado não pode ultrapassar 10 segundos.  
3. A resposta com o serviço (requisições internas) não pode ultrapassar 5 segundos.  
4. O sistema deve suportar no mínimo 5 trajetórias armazenadas simultaneamente sem degradação perceptível de desempenho.  
5. O sistema deve ser capaz de detectar perda de conexão com o carrinho.  
6. A conexão end-to-end entre o serviço e a ESP deve ser mantida de forma contínua.  
7. O software deve garantir integridade dos dados durante a comunicação e armazenamento.  
8. A interface deve ser intuitiva e responsiva, adaptando-se a mobile e desktop.  
9. As mensagens de erro e status devem ser claras e informativas, evitando termos técnicos desnecessários.  
10. O sistema deve permitir configuração dos parâmetros de comunicação com o carrinho sem necessidade de recompilar.  
11. O software deve validar as instruções antes de enviá-las ao carrinho, prevenindo comandos incorretos ou malformados.  
12. A interface deve ser compatível com os principais navegadores caso utilize uma camada web local.
13. A interface deve ser intuitiva e responsiva, adaptando-se a mobile e desktop.
